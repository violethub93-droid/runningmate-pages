## 2. 제품 개념

혼자 달리는 사람 옆에서 동행자처럼 말을 걸어주는 앱. 처음 구상한 세 가지 기능은 다음과 같았다.

1. G센서 + GPS로 러닝 코스·구간 목표 설정
2. 고도 데이터로 코스(경사·이동거리) 분석 → 실시간 페이스 개선(보폭·속도)
3. 일일 러닝 거리·운동량 측정

**핵심 통찰:** 위 세 기능은 전부 *측정·분석*이며, 그것만으로는 Strava·Nike Run Club과 차별화되지 않는다. "러닝메이트"라는 이름이 약속하는 **"메이트(동행자)"** 는 측정이 아니라 **실시간 음성 코치**에서 나온다. 그래서 제품의 정체성을 "AI 음성 코치"에 두기로 결정.

> 센서·분석·음성 출력은 표준 기술이라 라이브러리로 해결된다. 진짜 제품의 핵심은 **"무엇을, 언제, 어떤 말투로 말할지" 판단하는 코칭 엔진**이다.

## 3. 차별화 전략 — vs 런데이

런데이는 "혼자 뛰는 초보 러너를 위한 음성 코칭"을 한국 시장에서 이미 점유한 직접 경쟁자다. **정면으로 같은 간판을 걸면 진다.** 차별화는 런데이가 *구조적으로 약한 축*에서만 가능하며, 그 축이 곧 우리의 핵심 결정과 일치한다.

| 축 | 런데이 | 러닝메이트 (차별화 쐐기) |
|----|--------|------------------------|
| 코칭 방식 | **프로그램 기반** (정해진 플랜의 타이밍 멘트) | **실시간 적응형** (지금 내 페이스·케이던스 이탈에 즉시 반응) |
| 목소리 | 단일 '아저씨' 풀보이스 — *부담된다는 후기 다수* | **페르소나 선택**, TTS 캐싱으로 확장 용이 |
| 음악 | 강도별 자동 선곡 | **케이던스 BPM 동기화** (보행을 끌어올리는 코칭 도구) |
| 종료 후 | 스탬프·기록 | **생성형 AI 서사형 리포트** |

**정면 대결을 피할 영역** (런데이 본진 — 소모전 주의): 커뮤니티·크루·음성 응원 등 소셜(+"혼자" 콘셉트와 충돌), 24일 PT형 구조화 훈련 플랜.

> **결론: "초보용 음성 코칭 앱"으로는 못 이긴다(그건 런데이다). "실시간 적응형 + 내가 고른 메이트"로 이긴다.**

## 4. 확정된 핵심 결정

| 영역 | 결정 |
|------|------|
| **음성** | 합성 TTS. 단 실시간 호출이 아니라 **사전 합성 → 음성파일 캐싱 → 재생.** (비용·오프라인 동시 해결) |
| **페르소나** | 기본값 차분한 코치형. 추후 친구형·동기부여형 확장. 구조는 처음부터 다중 페르소나 대비. |
| **타깃** | 초보 러너 우선(관대한 코칭·부상예방·습관화). 추후 중급자용 기록 단축 모드. |
| **음악** | 음악 취향 연계보다, **코칭 없는 구간에 깔리는 케이던스 동기화 BGM** + 덕킹. |
| **페르소나 우선순위** | 코치형·친구형이 자연스러움(현장 체감). 동기부여형은 후순위. |

## 5. 핵심 설계

### 5.1 실시간 코칭 루프

```
[센서 수집] → [실시간 분석] → [코칭 엔진] → [음성 피드백]
 GPS·G센서·고도  페이스·보폭·경사   타이밍 판단    TTS 코칭
        ↑                                              │
        └───────── 사용자가 반응하면 다시 측정 ──────────┘
```

가운데 **코칭 엔진**이 제품의 정체성. 세 가지를 판단한다.
- **무엇을 말할지**: 페이스/케이던스/경사 이탈에 따른 조언
- **언제 말할지**: ①이탈 임계치 초과 ②주기적 체크인 ③마일스톤 — 이 트리거에서만 (잔소리 방지)
- **어떤 말투로**: 다그치지 않는 동행자 톤 (= 브랜드)

### 5.2 오디오 설계

**음성 (캐싱형 TTS)**
- 코칭 멘트는 유한한 문구 풀 → 실시간 TTS 불필요.
- 고품질 뉴럴 TTS로 **사전 1회 합성 → 음성파일 캐싱 → 재생만.** (비용은 문구당 1회, 오프라인 재생 가능)
- 생성형 AI는 ①문구 풀을 미리 다양하게 생성 ②종료 후 서사형 리포트 — 이 두 군데에만.
- `{n}킬로미터` 같은 동적 값은 범위(1~5km 등)를 미리 통문장으로 합성.

**음악 (케이던스 동기화 BGM)**
- 로열티프리 또는 직접 생성한 루프형 BGM (BPM 통제 가능).
- **덕킹**: 메이트 발화 시 BGM 감소 → 멘트 후 복귀.
- **BPM 전략**: 현재보다 약간 높은 BPM으로 케이던스를 자연스럽게 유도. 전환은 코칭 사이/km 구간에서만.

### 5.3 멘트 데이터 구조 (이중 축)

`상황(situation) × 페르소나(persona) → 멘트 변형 배열` + 레벨(초보/중급) 분기.

```json
{
  "situation_id": {
    "trigger": "발화 조건",
    "cooldown_sec": 45,
    "level": "beginner | intermediate | all",
    "variants": {
      "coach":     ["변형1", "변형2", ...],
      "friend":    ["..."],
      "motivator": ["..."]
    }
  }
}
```

- **페르소나(말투)와 레벨(내용)은 서로 다른 축** — 합치지 않는다.
- 발화 시: cooldown 체크 → 페르소나 풀에서 **직전과 다른 변형** 선택 → 재생.
- 페르소나 추가 = 배열만 채움(코드 변경 없음).

## 7. 로드맵

### Phase 0 — MVP (검증용 최소 단위)
실시간 페이스 추적 · 목표 페이스 설정 · 페이스 이탈 시 음성 코칭 1줄 · 종료 후 AI 요약 리포트 · 거리/시간/평균 페이스 기록.
**기본값:** 초보 타깃 · 코치형 · 캐싱형 TTS. (보폭·경사·코스추천·소셜은 제외)

### Phase 1 — 코칭 깊이
가속도계 케이던스·보폭 추정 · 케이던스 자세 코칭 · 발화 타이밍 정교화 · 코칭 없는 구간 BGM+덕킹 · 문구 풀 다양화 · 페르소나 데이터 구조 확정.

### Phase 2 — 지형·코스 지능
고도 데이터 경사 분석 · 경사 보정 페이스(GAP) · 코스/구간 목표 설정 · 오프라인 GPS 보정 · 일일/주간 운동량 대시보드.

### Phase 3 — 확장
페르소나 확장(친구형·동기부여형) · 케이던스 BPM 동기화 BGM · 중급자 모드(스플릿·레이스 페이스·GAP) · 코스 추천 · (신중히) 실시간 대화형 메이트.

### 우선순위 원칙
낮은 난이도 × 높은 우선순위부터. 핵심 가설(실시간 음성이 동행자처럼 느껴지는가)이 검증되기 전엔 중급 모드·고난도 기능을 쌓지 않는다.

## 8. 현재 단계 & 다음 액션

### 1차 현장 테스트 결과 (검증 완료)

클로바더빙 무료 음성을 **수동 재생**하며 실제로 달려 검증.

**✅ MVP 핵심 가설 통과** — "동행자처럼 느껴졌는가, 트래커 음성안내처럼 느껴졌는가?" → **동행자처럼 느껴짐.** 단, "특별한 경험"엔 음성 자연스러움이 부족.

**핵심 발견 (콘셉트가 아니라 완성도 과제):**
1. **GPS 페이스 노이즈 ↔ 지연.** 계단 한 층에도 페이스 급등. 누르면 적막, 빠르게 반응하면 오발사. → 중앙값 필터 + 경사 보정 + 케이던스/기압계.
2. **적막함 (가정 뒤집힘).** "잔소리"가 아니라 너무 조용한 게 문제. 페이스 변화에 멘트가 늦음. → cooldown·트리거를 존재감 채우는 방향으로.
3. **음성 자연스러움 부족.** → Pro 엔진 상향 또는 핵심 멘트 성우 녹음 하이브리드.
4. **문구 풀이 작아** 빈도·반복 판단 불가. → 코치형·친구형 풀 확장 + 자동 트리거 필요.

> **방향은 맞았다(동행자 확인). 이제 콘셉트가 아니라 완성도 문제.**

### 다음 액션 — 2차 현장 테스트 (자동 트리거)

1. **자동 트리거 프로토타입**(`runningmate_prototype_auto.html`)으로, 실제 폰 GPS를 받아 **페이스 변화에 맞춰 자동 발화**되는지 검증. (수동 재생의 지연 한계 제거)
2. 중앙값 스무딩이 **계단·노이즈 스파이크**를 잡아주는지 현장 확인 (원시 페이스 vs 스무딩 페이스 동시 표시).
3. **적막함이 해소**됐는지 — 체크인 간격·이탈 민감도 조정값이 적절한지 확인.
4. 다음 라운드 전 코치형·친구형 **문구 풀 확장**.

> 주의: GPS·음성을 폰에서 쓰려면 **HTTPS 환경 필요**(GitHub Pages·Netlify 등). 로컬 파일은 폰에서 위치권한이 막힐 수 있음.

## 9. 남은 열린 질문

- TTS 벤더 선정: 클로바(한국어 음질 최상) vs 구글(무료구간 넉넉) vs 타입캐스트·ElevenLabs(개성 보이스). 캐싱 방식이라 **비용보다 음질·다양성이 기준.**
- BGM: 로열티프리 라이브러리 vs 직접 생성 루프형 스템(BPM 통제·차별성↑, 제작 공수↑).
- 초보→중급 전환 트리거: 수동 선택 vs 기록 기반 앱 제안.
- 페르소나별로 TTS 목소리도 다르게 줄지 (다양성↑, 사전 합성 세트 배수 증가).
